# ページ別 JS 運用ルール

`src/assets/js/pages/` はページ専用の JS を配置するフォルダです。

---

## フォルダの役割

| 環境 | 動作 |
|------|------|
| 開発（`npm run dev`） | `index.js` が `import.meta.glob('./pages/*.js')` で全ファイルを動的ロード（HMR有効） |
| 本番（`npm run build`） | `tools/build-js-bundle.js` が全ファイルを `script.js` に自動結合 |

---

## ファイル命名規則

**ページ名（HTMLファイル名）と一致させる。**

```
index.html  →  pages/top.js     ※ トップページは "top" とする
about.html  →  pages/about.js
service.html →  pages/service.js
```

---

## ファイルの書き方

### 基本構造

ページ固有の要素セレクターで存在チェックを行い、対象外ページでは即 `return` する。

```js
// ============================================================
// ○○ページ専用 JS
// 対象: .p-○○-fv が存在するページ（○○.html）
// ============================================================

(function() {
    if (!document.querySelector(".p-○○-fv")) return;

    // ここにページ専用の処理を書く
})();
```

### ポイント

- **即時関数（IIFE）で変数スコープを閉じる** — グローバル汚染を防ぐ
- **先頭に存在チェックを入れる** — 対象外ページでエラーが出ないようにする
- `gsap` / `jQuery($)` / `Swiper` はグローバルに公開済みのため、`import` 不要

---

## 担当ルール（チーム開発）

- **自分の担当ページの JS ファイルのみ作成・編集する**
- 他人のページ JS は触らない
- `script.js`（共通 JS）を変更する場合は**事前に共有**する

---

## 新規ページを追加する手順

1. `src/assets/js/pages/ページ名.js` を新規作成
2. 先頭に存在チェックを書く
3. `npm run dev` を再起動すると自動で読み込まれる

> `_index.scss` と異なり、ページ別 JS は自動生成ではなく**手動で作成**する。

---

## よくある間違い

| NG | OK |
|----|----|
| `script.js` にページ専用処理を書く | `pages/ページ名.js` に書く |
| 存在チェックなしで DOM 操作する | `if (!document.querySelector("...")) return;` を先頭に入れる |
| `import` でライブラリを読み込む | グローバル変数（`gsap` / `$` / `Swiper`）をそのまま使う |
| ファイル名をページ名と別にする | HTML ファイル名と統一する |
